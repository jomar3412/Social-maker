# Knowledge Base Index

This directory contains all context documents used by the Content Engine for generation, validation, and learning.

## Structure

```
knowledge_base/
├── external/           # User-provided documents (immutable by engine)
│   ├── specs/          # Authoritative specifications and rules
│   ├── transcripts/    # YouTube transcripts, interviews, etc.
│   └── research/       # Research notes, frameworks, guides
├── internal/           # Engine-generated knowledge from real runs
│   ├── insights/       # Learning summaries and observations
│   ├── patterns/       # Discovered repeat patterns (good and bad)
│   └── rules/          # Rule overrides based on performance data
├── index.md            # This file (human-readable index)
└── manifest.json       # Machine-readable registry of all documents
```

## How Documents Are Used

1. **At Run Start**: The engine loads relevant external specs and internal insights
2. **During Generation**: Dopamine Ladder rules are enforced; internal patterns inform optimization
3. **After Feedback**: New insights are extracted and compared against external specs
4. **Conflicts**: When internal data contradicts external specs, both are preserved; internal is preferred only when sample size threshold is met (default: 10 runs)

## Adding New Documents

### Adding an External Document

1. Place the file in the appropriate subfolder:
   - `external/specs/` for authoritative rules
   - `external/transcripts/` for transcribed content
   - `external/research/` for research notes

2. Add an entry to `manifest.json`:
   ```json
   {
     "id": "unique_id",
     "type": "spec|transcript|research",
     "scope": "external",
     "path": "external/subfolder/filename.md",
     "title": "Human-readable title",
     "description": "What this document contains",
     "source": "user_provided|youtube|article|etc",
     "added_at": "YYYY-MM-DD",
     "tags": ["relevant", "tags"],
     "priority": "authoritative|reference|supplemental"
   }
   ```

3. Update this index.md with a row in the appropriate table.

### Internal Documents (Auto-Generated)

Internal documents are created and updated automatically by the engine. Do not manually edit them.

---

## Conflict Resolution Policy

When internal learning contradicts external specs:

- **External specs are never overwritten**
- Conflicts are recorded in `internal/insights/conflicts.md`
- Internal data is preferred when:
  - Sample size >= threshold (default 10 runs for niche/style combo)
  - The internal pattern is statistically significant
- Otherwise, external spec takes precedence

## Machine-Readable Registry

For programmatic access, use `manifest.json`. It contains:
- All document metadata
- Section indexes for quick lookup
- Configuration for learning thresholds
- Internal insight/pattern/rule registries
